# ctx — Product Guidelines

## Tone of Voice
The `ctx` project uses a **Technical and Precise** tone. All communications, whether from the AI, in code comments, or in documentation, should prioritize accuracy, authority, and technical depth.

## UX Principles
The user experience for `ctx` is built on three core pillars:
- **Efficiency for Power Users:** Optimized for rapid command execution, keyboard shortcuts, and CLI-first workflows.
- **Clarity and Discoverability:** Ensures that workflows are intuitive and that functionality is easy to find and understand.
- **Reliability and Predictability:** Focuses on stable and expected behavior across all integrated AI providers, building trust with the user.

## Naming Conventions
To maintain a clean and consistent project structure, all new features, scripts, and components must follow the **Kebab-case (Standard)** convention.
- **Files/Directories:** `ctx-new-feature`, `scripts/new-tool.js`
- **Prefixes:** Use the `ctx-` prefix for core tools and components to ensure they are easily identifiable within the ecosystem.

## Documentation Style
The project follows a **Markdown-First Structure**.
- All internal and external documentation should be written in Markdown.
- Use a clear, hierarchical structure with descriptive headers and lists.
- Prioritize technical depth while maintaining readability.
