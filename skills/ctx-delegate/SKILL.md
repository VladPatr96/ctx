# /ctx-delegate — Smart Task Delegation

Автоматический роутинг задач к лучшему AI провайдеру.

## Использование

```
/ctx-delegate <задача>                     — авто-роутинг
/ctx-delegate --provider gemini <задача>   — явный выбор провайдера
/ctx-delegate --multi <задача>             — параллельно нескольким провайдерам
```

## Маршрутизация

| Тип задачи | Провайдер | Почему |
|------------|-----------|--------|
| code review, ревью | codex | Sandbox + изолированный анализ |
| анализ, аудит, весь проект | gemini | 1M+ контекст |
| документация, readme, docs | gemini | Большой контекст для всего кода |
| exec, запусти, скрипт | codex | Sandbox execution |
| перевод, i18n, локализация | gemini | Все файлы в одном контексте |
| рефакторинг, оптимизация | codex | Sandbox + diff apply |
| план, архитектура | claude | Plan mode + Task tool |
| workflow, pipeline | claude | Native agents + hooks |
| JSON, structured output | opencode | --format json |
| сравнение моделей | opencode | Мульти-модельность |

## Workflow

### Шаг 1: PARSE

Разбери команду пользователя:

1. Если есть `--provider <name>` — извлеки имя провайдера, режим = explicit
2. Если есть `--multi` — режим = multi
3. Иначе — режим = auto

Извлеки текст задачи (всё после флагов).

### Шаг 2: ROUTE

Определи лучшего провайдера. Используй Task tool с subagent_type "Bash":

```bash
node -e "
import { route, routeMulti } from '${CLAUDE_PLUGIN_ROOT}/scripts/providers/router.js';
const task = process.argv[1];
const result = JSON.stringify({ single: route(task), multi: routeMulti(task) });
console.log(result);
" "<текст задачи>"
```

Покажи пользователю рекомендацию:

```
## Рекомендация роутера

**Задача:** <текст задачи>
**Провайдер:** <имя> (confidence: <0-1>)
**Причина:** <описание из bestFor>
```

Если `--multi`, покажи всех кандидатов.

### Шаг 3: CONFIRM

Спроси подтверждение у пользователя (через AskUserQuestion):

- Вариант 1: Принять рекомендацию
- Вариант 2: Выбрать другого провайдера
- Вариант 3: Отменить

Если пользователь передал `--provider`, пропусти подтверждение.

### Шаг 4: DISPATCH

Вызови провайдера в зависимости от режима:

**Explicit / Auto (один провайдер):**

Используй Task tool с subagent_type "Bash":

Для Gemini:
```bash
gemini -p "<задача>" -o text 2>&1 | head -200
```

Для Codex:
```bash
codex exec --ephemeral --skip-git-repo-check "<задача>" 2>&1 | head -200
```

Для OpenCode:
```bash
opencode run "<задача>" 2>&1 | head -200
```

Для Claude (через Task tool):
```
Используй Task tool с subagent_type: "general-purpose" и model: "sonnet"
```

**Multi (несколько провайдеров параллельно):**

Запусти всех кандидатов из routeMulti **параллельно** (несколько Tool calls).
Таймаут: 60 секунд на провайдера.

### Шаг 5: RESULT

Покажи результат:

```
## Результат делегирования

**Задача:** <текст>
**Провайдер:** <имя> (routing: <auto|explicit|multi>)
**Статус:** success | error

### Ответ
<ответ провайдера>
```

Для multi-режима покажи таблицу:

```
| Провайдер | Статус | Ответ (кратко) |
|-----------|--------|----------------|
| ...       | ...    | ...            |
```

### Шаг 6: LOG

Запиши результат через MCP tool `ctx_log_action`:

```
action: "delegate"
file: "<провайдер>"
result: "<статус>: <краткое описание результата>"
```

## Правила

1. **Fallback** — если рекомендованный провайдер упал (circuit breaker / ошибка), попробуй следующего по приоритету
2. **Таймаут** — 60 секунд на провайдера, не ждать дольше
3. **Изоляция** — каждый провайдер получает только задачу, не видит ответы других
4. **Прозрачность** — всегда показывай почему выбран конкретный провайдер
5. **Без дефолта** — если роутер не нашёл паттерн, спроси пользователя вместо молчаливого fallback
